#### 第一步：立即运行以下命令

```bash
git ls-files | cut -d/ -f1 | sort -u
```

> 命令的输出结果可以作为很好的参考，特别注意项目配置文件和lock文件。

#### 第二步：启用相关Skill

1. 启用`planning-with-files`等任何时候都启用的Skill。

2. 查看所有现有Skill的介绍，然后自动启用符合当前项目语言/配置/工具链等的所有Skill。

3. 在此之后，如果发现有未启用的符合项目的Skill，同样自动启用。

例：发现`Cargo.lock`启用`rust`相关Skill，发现`uv.lock`启用`uv`和`python`相关Skill。

## 探索

- 当遇到探索任务时，使用`explore`这个Agent。建议优先使用已启用的MCP来获取信息。

## 编码

- 尽量通过负面条件前移的方式，减少代码嵌套。
- 避免大块编辑。对于现有文件，避免一次性编辑10行以上。
- 避免回退修改。

## 注释

- 各个函数和类型，必须添加文档注释。
- 对于复杂实现，添加注释以方便审阅。
- 注释语言必须跟随代码库已有注释。例：代码库中原先的注释使用英文，则新增注释也应该使用英文。

## 命令行

### 推荐

- `fd`：查询文件。
- `rg`：查询文件内容。

### 低优先级

- `sd`：改为使用编辑工具。
- `sed`：改为使用编辑工具。

### 极低优先级（不推荐）

- `cd`：如果在当前项目根目录下，完全不需要。
- `echo`：编辑文件改为使用编辑工具。截断输出改为使用`head`或`tail`。
- `cat`：完全不需要。
