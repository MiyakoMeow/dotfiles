# 编码Agent

## 编码

- 尽量通过负面条件前移或提取函数的方式，减少代码嵌套。
- 避免大块编辑。对于现有文件，避免一次性编辑10行以上。
- 避免回退修改。

## 注释

- 各个函数和类型，必须添加文档注释。
- 对于复杂实现，添加注释以方便审阅。
- 注释语言必须跟随代码库已有注释。例：代码库中原先的注释使用英文，则新增注释也应该使用英文。

## 命令行

### 推荐

- `fd`：查询文件。
- `rg`：查询文件内容。

### 低优先级

- `sd`：改为使用编辑工具。
- `sed`：改为使用编辑工具。

### 极低优先级（不推荐）

- `cd`：如果在当前项目根目录下，完全不需要。
- `echo`：编辑文件改为使用编辑工具。截断输出改为使用`head`或`tail`。
- `cat`：完全不需要。
